## Linked List : Introduction

### What is a Linked List?

Imagine you are given a task to maintain data of cars entering a parking lot.  
Since the number of cars may change daily, using a fixed-size data structure like an array might not be helpful.  
This is where a linked list becomes useful, as it allows easy addition and removal of elements.

A Linked List is a linear data structure that can be visualized as a chain of nodes,  
where each node represents an element and is connected to the next node.

Unlike arrays, elements in a linked list are not stored in contiguous memory locations.

---

### Arrays vs Linked Lists

For arrays, adding a new element requires the next memory location to be empty.  
Since memory allocation is fixed after initialization, resizing an array dynamically is difficult.

Linked lists overcome this limitation by allocating memory dynamically for each new node.

---

### Structure of a Linked List Node

A linked list node contains two main parts:

1. Data → stores the value of the element  
2. Pointer (or reference) → stores the address of the next node  

---

### Important Terms

Head → The first node of the linked list  
Tail → The last node of the linked list (points to null)
